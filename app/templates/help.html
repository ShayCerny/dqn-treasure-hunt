Help & Configuration Guide

How It Works

This app trains a Deep Q-Network (DQN) agent to navigate a maze and find hidden treasure. The agent learns entirely through trial and error — it starts by making random moves and gradually figures out which actions lead to the reward.

The training loop
  1. Explore: The agent picks a random starting cell and moves through the maze, choosing actions randomly (exploration) or based on its current knowledge (exploitation).
  2. Remember: Each move — the state before, action taken, reward received, and resulting state — is stored in a replay memory buffer.
  3. Learn: A prioritized batch of past experiences is sampled from memory — transitions the network currently finds surprising are replayed more often. The network's weights are updated to better predict which actions lead to higher rewards.
  4. Repeat: This continues for every epoch. The agent's behavior improves as its Q-values (estimated action values) become more accurate.
Algorithm improvements

Double DQN reduces a well-known overestimation bias in standard DQN. Vanilla DQN uses the same (target) network to both select the best next action and estimate its value, which inflates Q-values over time. Double DQN decouples these roles: the main network selects the best action, and the target network evaluates it. This small change produces more accurate value estimates and a more stable policy.

Prioritized Experience Replay (PER) replaces uniform random sampling with sampling proportional to each transition's TD error — the gap between the predicted and target Q-value. Transitions the network found surprising are replayed more often, leading to faster convergence. A sum-tree data structure keeps sampling efficient (O(log n)). Importance-sampling (IS) weights correct the bias introduced by non-uniform sampling, with the correction annealed from βstart to 1.0 over the course of training.

At the end of training you can watch a replay of the agent navigating the maze with the weights it learned.

Configuration Reference
SettingDescriptionGuidance
Epochs Total number of training cycles. Each epoch the agent plays one full game and updates the network. Start with 500–1000. Increase if the agent hasn't converged by the end.
Batch Size Number of past experiences randomly sampled from memory and used for one network update step. 32 is a solid default. Larger batches are more stable but slower per step.
Replay Memory Maximum number of past (state, action, reward, next state) tuples stored. Older entries are discarded when full. 512–1000 works well for small mazes. Larger mazes may benefit from more memory.
Learning Rate Controls how much the network weights shift after each batch update. Higher = faster learning but more instability. 0.001 is the standard Adam default. Try 0.0005 if training is erratic.
Hidden Layer Size Number of neurons in each of the two hidden layers of the Q-network. More neurons = more capacity to learn complex patterns. 64 matches the default 8×8 maze size. Larger mazes may benefit from 128 or more.
Epsilon Start The agent's initial exploration rate. At 1.0 the agent acts completely randomly, ignoring its Q-values entirely. Always start at or near 1.0 so the agent builds a diverse memory before exploiting.
Epsilon Min The floor for the exploration rate. Even a fully trained agent will make a small percentage of random moves. 0.05 (5%) is a common choice. Lower values let the agent exploit more aggressively late in training.
Epsilon Decay Multiplicative factor applied to epsilon each epoch. Epsilon shrinks from its starting value toward Epsilon Min over time. 0.995 decays slowly over ~1000 epochs. Use 0.99 for faster decay with fewer epochs.
Patience Number of consecutive epochs where the win rate must stay at 100% before training stops automatically (early stopping). 10 is a good default. Raise it if you want more confidence in convergence.
Early Stop Loss Delta Minimum relative improvement in loss over the patience window required before early stopping can trigger. Measured as (max − min) / max within the window. 0.1 works well in most cases. Lower values demand tighter convergence before stopping.
PER Priority Exponent (α) Controls how strongly transitions are prioritized by their TD error. α = 0 gives uniform sampling (standard experience replay); α = 1 gives fully proportional prioritization. 0.6 is the value from the original PER paper and works well in practice.
IS Correction Start (β) Starting value for importance-sampling weights that correct the bias introduced by prioritized sampling. Annealed to 1.0 over the course of training. 0.4 is the standard starting value. Higher values correct bias more aggressively early on.
Target Update Freq How many epochs between copying the main network's weights into the slower-moving target network used for computing Q-targets. 50 is stable for most configurations. Lower values track changes faster but can destabilise training.
Discount Factor How much the agent values future rewards compared to immediate ones. 0.95 means a reward 10 steps away is worth 0.9510 ≈ 60% of an immediate reward. 0.9–0.99 works well for most maze sizes. Higher values encourage longer-term planning.
Quick Tips
  • If the win rate stays at 0% for hundreds of epochs, try increasing the number of epochs or slowing epsilon decay so the agent explores longer.
  • If training is unstable (loss oscillates wildly), lower the learning rate or increase the replay memory size.
  • The Completion Check on the Run History page re-runs the agent from every free cell in the maze — it passes only if the agent wins from all positions.
  • Larger mazes generally need more epochs, a larger replay memory, and possibly a larger hidden layer size.
  • You can cancel training at any time and still watch a replay — the weights at the point of cancellation are saved.
